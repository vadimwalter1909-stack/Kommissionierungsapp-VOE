# 📘 **VOE – Komm‑App Projekt‑Prompt (Stand: aktuell)**

## 🧩 **Projektbeschreibung**
Die VOE‑Komm‑App ist eine **FastAPI‑basierte Web‑Applikation** zur Steuerung von:

- **Produktion**
- **Logistik**
- **Kommissionierung**
- **Parkzonen‑Handling**
- **Arbeitsplatz‑Artikel (neu)**

Sie nutzt:

- **FastAPI** (Backend)
- **Jinja2** (Templates)
- **SQLite** (Datenbank)
- **JSON‑Dateien** für Zusatzdaten
- **Railway‑Deployment** (später)
- **OCR‑Modul** für Etiketten‑Erkennung

---

## 🗂 **Wichtige Dateien & Module**

### **Backend**
```
backend/
 ├─ main.py                 → App‑Setup, Middleware, Routing
 ├─ routes/
 │    ├─ logistik.py        → Logistik‑Übersicht, Detailseite, Arbeitsplatz‑Artikel‑Routen
 │    ├─ produktion.py      → Produktionslogik
 │    └─ auth/              → Login‑Middleware
 ├─ utils/
 │    ├─ arbeitsplatz_loader.py  → NEU: lädt Arbeitsplatz‑Artikel

 └─ data/
      ├─ arbeitsplatz_artikel.json  → NEU: gespeicherte Arbeitsplatz‑Artikel
      └─ weitere JSON/DB Dateien
```

# 🧠 **Wichtige Logikpunkte**

### 1. **Arbeitsplatz‑Artikel beeinflussen Logistik‑Detailseite**
Wenn `artikel_nr` in `arbeitsplatz_artikel.json` steht:

```python
"liegt_am_arbeitsplatz": True
```

→ Kommissionierung entfällt  
→ UI zeigt Hinweis

---

### 2. **+‑Button in logistik.html**
Steht rechts neben dem OCR‑Button:

```html
<a href="/arbeitsplatz-artikel">+</a>
```

---

### 3. **Login‑System**
- Login‑Seite blockiert alle anderen Seiten, bis man eingeloggt ist  
- Admin‑Rolle erlaubt Löschen von Aufträgen
